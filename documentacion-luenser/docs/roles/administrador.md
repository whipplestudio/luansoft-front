# Documentación para Usuarios con Rol de Administrador

## Introducción

Este documento describe los módulos y funcionalidades disponibles para los usuarios con el rol de **Administrador** en el sistema de gestión fiscal de Luenser. El rol de Administrador está diseñado para proporcionar acceso completo a todas las funcionalidades del sistema, permitiendo la gestión integral de la plataforma, usuarios, datos y configuraciones.

## Responsabilidades del Administrador

Como Administrador del sistema, usted es responsable de:

1. **Gestión de usuarios**: Crear, modificar y eliminar cuentas de usuario, asignar roles y permisos.
2. **Configuración del sistema**: Mantener actualizadas las configuraciones generales y específicas.
3. **Mantenimiento de datos maestros**: Gestionar catálogos, regímenes fiscales y otros datos de referencia.
4. **Supervisión de operaciones**: Monitorear el uso del sistema y resolver incidencias.
5. **Respaldo y seguridad**: Asegurar la integridad y disponibilidad de los datos.

## Módulos Accesibles

### 1. Dashboard

**Ubicación:** Página principal (inicio) del sistema

**Descripción:** El Dashboard proporciona una visión general del estado del sistema y métricas clave.

### 2. Usuarios

**Ubicación:** Menú principal > Usuarios

**Descripción:** Gestión completa de usuarios del sistema.

**Funcionalidades disponibles:**
- Crear nuevos usuarios
- Asignar roles y permisos específicos
- Modificar información de usuarios existentes
- Desactivar o eliminar usuarios
- Restablecer contraseñas
- Ver historial de actividad de usuarios

**Flujo de trabajo para crear un nuevo usuario:**
1. Acceder al módulo de Usuarios
2. Hacer clic en "Nuevo Usuario"
3. Completar la información requerida (nombre, correo, rol, etc.)
4. Asignar el rol correspondiente
5. Guardar los cambios
6. El sistema enviará credenciales al nuevo usuario

### 3. Contadores

**Ubicación:** Menú principal > Contadores

**Descripción:** Gestión de contadores y sus asignaciones.

**Funcionalidades disponibles:**
- Registrar nuevos contadores
- Editar información de contadores existentes
- Asignar clientes a contadores
- Ver carga de trabajo y estadísticas de desempeño
- Desactivar o eliminar contadores

### 4. Clientes

**Ubicación:** Menú principal > Clientes

**Descripción:** Administración completa de la información de clientes.

**Funcionalidades disponibles:**
- Registrar nuevos clientes
- Editar información de clientes existentes
- Asignar regímenes fiscales
- Gestionar documentos y expedientes
- Ver historial de procesos y actividades
- Desactivar o eliminar clientes

### 5. Contactos

**Ubicación:** Menú principal > Contactos

**Descripción:** Gestión de contactos asociados a clientes.

- Registrar nuevos contactos
- Editar información de contactos existentes

### 6. Asignación de Contadores

**Descripción:** Gestión de la relación entre contadores y clientes.

**Funcionalidades disponibles:**

### 7. Asignación de Contactos

**Ubicación:** Menú principal > Regímenes Fiscales

**Descripción:** Gestión de los diferentes regímenes fiscales aplicables.

**Descripción:** Registro histórico de todos los procesos realizados.

- Consultar procesos completados
- Filtrar por cliente, tipo, fecha, etc.

## Mejores Prácticas para Administradores

1. **Gestión de usuarios:**
   - Asigne roles según el principio de mínimo privilegio
   - Revise periódicamente los permisos de usuarios
   - Desactive cuentas inactivas o no utilizadas
   - Implemente políticas de contraseñas seguras

2. **Mantenimiento del sistema:**
   - Realice respaldos regulares
   - Verifique la integridad de los datos periódicamente
   - Mantenga actualizados los catálogos y datos maestros
   - Monitoree el rendimiento del sistema

3. **Seguridad:**
   - Revise regularmente el registro de actividades
   - Implemente autenticación de dos factores para cuentas críticas
   - Establezca políticas de bloqueo por intentos fallidos
   - Capacite a los usuarios sobre prácticas seguras

4. **Gestión de datos:**
   - Implemente políticas de retención de datos
   - Realice limpieza periódica de datos obsoletos
   - Verifique la calidad de los datos ingresados
   - Documente los cambios importantes en la estructura de datos

## Preguntas Frecuentes

**P: ¿Cómo puedo otorgar permisos temporales a un usuario?**
R: Actualmente, el sistema no admite permisos temporales. Se recomienda crear un rol específico con los permisos necesarios y asignarlo temporalmente al usuario, cambiándolo nuevamente cuando ya no sea necesario.

**P: ¿Puedo tener múltiples administradores en el sistema?**
R: Sí, puede haber múltiples usuarios con rol de Administrador. Sin embargo, se recomienda limitar este número por razones de seguridad y control.

**P: ¿Cómo puedo ver quién realizó cambios específicos en el sistema?**
R: Acceda al Registro de Actividad en la sección de Configuración. Allí podrá filtrar por tipo de acción, fecha y usuario para identificar quién realizó cambios específicos.

**P: ¿Qué debo hacer si un usuario olvida su contraseña?**
R: En el módulo de Usuarios, busque al usuario afectado y utilice la opción "Restablecer contraseña". El sistema enviará un correo electrónico al usuario con instrucciones para crear una nueva contraseña.

**P: ¿Cómo puedo crear un nuevo tipo de proceso?**
R: Acceda al módulo de Procesos, seleccione "Nuevo tipo de proceso", complete la información requerida incluyendo etapas, plazos y requisitos, y guarde los cambios.
